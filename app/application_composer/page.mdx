<br></br>

Application Composer is a configuration tool in **Oracle Fusion Sales & Service Cloud** used to **extend and customize CRM applications** in a controlled and upgrade-safe way.

It allows administrators and developers to configure business behavior **without writing heavy backend code**.

---

## What is Application Composer?

Application Composer is used to:

- Configure **standard business objects**
- Create **custom objects**
- Define **relationships** between objects
- Add **business logic** using rules and Groovy
- Control **security and roles**
- Configure **integrations**

All changes are stored as **metadata**, not source code.

---

## Main Modules in Application Composer

Application Composer is divided into multiple functional areas:

- **Objects** – Define and manage business entities
- **Relationships** – Connect objects together
- **Object Workflows** – Automate business logic
- **Global Functions** – Reusable Groovy logic
- **Security** – Role and privilege control
- **Web Services** – Integration endpoints
- **Smart Actions & Runtime Messages** – UI and runtime behavior

Each module focuses on a specific aspect of application configuration.

---

## Who Uses Application Composer?

- **Administrators** – Configuration and security
- **Functional Consultants** – Business rules and workflows
- **Technical Developers** – Groovy scripts and integrations

---

## What Application Composer Is NOT Used For

Application Composer is not used to:

- Build full applications
- Modify Oracle-delivered database tables
- Write unrestricted backend code

It is designed for **safe, controlled customization**.

---

## Summary

Application Composer is the central customization tool in Oracle Fusion that allows teams to:

- Extend CRM functionality
- Control business behavior
- Maintain upgrade safety
- Avoid custom code complexity
