# Oracle Fusion Sales & Service Cloud

Oracle Fusion Sales and Service Cloud is part of **Oracle Fusion Cloud Applications**.  
It helps organizations manage **customers, sales activities, and customer support** using a cloud-based system.

This page provides a **high-level understanding** of how the platform works and what it is used for.

---

## What is Oracle Fusion Cloud (OFC)?

**Oracle Fusion Cloud (OFC)** is a **cloud-based enterprise platform** provided by Oracle to manage business processes.

It runs completely in a **web browser** and is maintained by Oracle, including upgrades and security.

### What is it used for?

Oracle Fusion Cloud is used to:

- Manage customer data
- Sell products and services
- Handle customer issues
- Automate business processes

**Learn More**: [Explore Oracle Fusion Cloud Features ‚Üí](https://docs.oracle.com/en/cloud/saas/applications-common/25d/oacex/index.html)
</Callout>

## Oracle Fusion Sales & Service Cloud

Sales & Service Cloud is the **CRM (Customer Relationship Management)** part of Oracle Fusion.

It is divided into two main areas:

- **Sales Cloud** ‚Äì for selling
- **Service Cloud** ‚Äì for customer support

Each area supports a different stage of the customer lifecycle.

## Sales Cloud

### What does Sales Cloud do?

Sales Cloud helps sales teams:

- Track potential customers
- Manage customer accounts
- Follow sales opportunities
- Record sales activities

It is mainly used by **sales users**.

### Common Sales Cloud Objects

| Object | Purpose |
|--------|---------|
| **Leads** | Potential customers |
| **Accounts** | Companies |
| **Contacts** | People |
| **Opportunities** | Possible sales deals |
| **Activities** | Calls, meetings, tasks |

## Objects in Oracle Fusion

An **Object** is a place where business data is stored.

You can think of it as:
- A table in a database
- A business entity in the application

Objects store data such as customers, orders, or service requests.

### Object Components

Each object can include:

| Component | Description |
|-----------|-------------|
| **Fields** | Data values |
| **Records** | Rows of data |
| **Pages** | UI screens |
| **Business Rules** | Validation and automation logic |
| **Security Settings** | Access control |

<Callout type="warning">
**Important**: Object customization requires proper understanding of data relationships and business processes.
</Callout>

<Callout type="info">
**Configuration**: [Configure Objects in Application Composer ‚Üí](https://docs.oracle.com/en/cloud/saas/applications-common/25d/oacex/objects.html)

## Customization Using Groovy Script

Oracle Fusion allows limited scripting using **Groovy**.

Groovy is used to add **business logic**, not to build full applications.

### What can Groovy do?

- ‚úÖ Validate data
- ‚úÖ Auto-update fields
- ‚úÖ Control behavior during save or update
- ‚úÖ Perform calculations

<Callout type="warning">
Groovy always runs **inside Oracle Fusion** and has limited scope. It cannot be used for external integrations.
</Callout>

### Where Groovy is Used

| Area | Purpose | Use Case |
|------|---------|----------|
| **Field level** | Validation | Check if email format is correct |
| **Object level** | Triggers | Auto-populate fields on record creation |
| **Global level** | Reusable functions | Shared calculation logic |

<Callout type="info">
**Learn More**: [Implement Groovy Logic ‚Üí](https://docs.oracle.com/en/cloud/saas/applications-common/25d/oacex/groovy-scripting.html)
</Callout>
